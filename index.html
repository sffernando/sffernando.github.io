iOS知识备忘

(一)UITextView没有自动滚动

当自定义一个自适应限高的UITextView的时候，当输入的文本内容超过textView最大的高度的时候，textView不会自己滚到最下面的输入指示闪烁光标。举个栗子，微信(测试机型: iPhone 6s，测试系统iOS 10.2.1， 测试版本: WeChat 6.5.5及之前的版本)输入框中连续点击输入自带表情，当超过5行之后继续点，则表情数量继续增加，但是文本内容不会向下滚，只有当删除一个之后，才会重新滚到最下面。
解决方案

这个设置了UITextView中的layoutManager(NSLayoutManager)是否非连续性布局属性。默认是YES,设置成NO之后UITextView就不会重置滑动了，就达到了我们想让其自己滚动的目的。

(二)本地图片的加载方式

基本上载项目中加载本地图片会使用imageNamed:方法，比如[UIImage imageNamed:@"1.png"]，其实还有另外一种方式:iOS中从程序bundle中加载UIImage一般有两种方法

  • 经常用的imageNamed:
  • 不怎么使用的:imageWithContentsOfFile:

这里说的比较明确了:

imageNamed:特点:

  1. 加载内存当中之后，会一直停留在内存当中，不会随着对象的销毁而销毁。
  2. 相同的图片，图片不会重复加载。

imageWithContentsOfFile:特点:

  1. 不会缓存，相同的图片会被重复加载内存当中。
  2. 对象销毁的时候，加载到内存中图片会随着一起销毁。

所以这两个方法的本质的区别就是缓存。基本上项目中一般的icon加载用imageNamed:没毛病，但是如果要加载一张比较大的图片或者只使用一次的图片(比如App中的引导图，用户交互指引图)的时候，应该用imageWithContentsOfFile:。

(三)install cocoapods

当更新或者安装cocoapods的时候，一般在终端中执行:sudo gem install cocoapods.

但是在最新的OS X系统中会出现这样的错误:

ERROR: While executing gem ... (Errno::EPERM)
Operation not permitted - /usr/bin/pod

解决方法: 重新执行sudo gem install -n /usr/local/bin cocoapods

iOS触摸事件的分发和响应链(二)

事件的响应链(The Responder Chain)

响应者对象(The Responder Object)

响应者链由响应者对象组成,响应者对象都是UIResponder的子类。

许多类型的事件依赖于用于事件传递的响应者链。响应者链是一系列链接的响应者对象,它从第一个响应者(responder)开始，并以应用程序对象(application object)结束。如果第一响应者不能处理事件，它将事件转发到响应者链中的下一个响应者。
